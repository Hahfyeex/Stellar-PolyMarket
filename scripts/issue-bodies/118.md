## Context
Gamification drives engagement. Users who consistently make accurate predictions should be visually recognized with a badge status system.

## Implementation Guide
1. Design 4 SVG badge files: bronze.svg, silver.svg, gold.svg, diamond.svg in /public/badges/
2. Create a getBadgeTier(marketsCount, accuracyPct) utility function with tier thresholds
3. Build a ReputationBadge component that renders the correct SVG based on tier
4. Add CSS box-shadow glow on hover — each tier has a unique glow color
5. Fetch user stats from GET /api/users/:wallet/stats and compute tier on the frontend
6. Render badge on profile page header and as a small icon in leaderboard rows

## Guidelines
- 4 SVG badge tiers: Bronze, Silver, Gold, Diamond
- Award logic based on prediction accuracy and total markets participated
- Distinct CSS box-shadow glow on hover per tier
- Thresholds: Bronze 10+ markets, Silver 50+ with 55% accuracy, Gold 100+ with 65%, Diamond 200+ with 75%
- Key requirement: SVG Assets / Glow Micro-interactions
