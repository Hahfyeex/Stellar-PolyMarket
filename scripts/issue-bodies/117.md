## Context
Traders need to quickly assess where liquidity is concentrated. A visual heatmap overlay gives instant depth perception without reading raw numbers.

## Implementation Guide
1. Fetch pool depth data per outcome from `GET /api/markets/:id`
2. Calculate each outcome's share as a percentage of total pool
3. Map percentage to CSS opacity: 0-20% = 0.1, 20-50% = 0.3, 50-80% = 0.6, 80-100% = 0.9
4. Render a `div` overlay behind each order book row using `position: absolute`
5. YES-side rows use `rgba(59, 130, 246, opacity)` (blue), NO-side use `rgba(249, 115, 22, opacity)` (orange)
6. Re-render overlay when pool data updates via WebSocket or polling

## Guidelines
- Layered CSS div overlays with opacity tied to pool size from the API
- Deeper liquidity zones appear more saturated
- Color scale: blue for YES-side, orange for NO-side depth
- Overlay updates when pool data refreshes
- Key requirement: CSS Depth Overlays / Transparency
