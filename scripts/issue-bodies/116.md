## Context
Users can only place one bet at a time requiring multiple wallet approvals. This creates friction especially on mobile where UX should feel native and fluid.

## Implementation Guide
1. Create a `BettingSlip` component with a slide-up drawer using CSS `transform: translateY`
2. Use Redux or Context to manage `isOpen`, `bets[]`, `addBet` and `removeBet` actions
3. Each queued bet shows: market title, selected outcome, amount, and a remove button
4. On submit, call `useBatchTransaction` to bundle all bets into one Freighter approval
5. Limit queue to 5 items — show a toast warning when user tries to add a 6th
6. On desktop (min-width: 1024px), render as a fixed right-side panel instead of a drawer

## Guidelines
- Slide-up drawer from bottom on mobile, side panel on desktop
- Queue up to 5 bets, bundle into single Freighter transaction
- Global drawer state via Redux or Context
- Bet summary per queued item with remove option
- Key requirement: Slide-up Drawer / Batching Logic
